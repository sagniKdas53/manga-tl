Problem

One shallow module owns transport, state, and pub/sub; its unmemoized interface leaks every AppContent render into four consumers, and a provider remount silently kills the SSE stream.

Solution

Deepen: transport behind one seam, list state behind another, job-event pub/sub behind a third — each interface becomes its own test surface.

Wins

  • locality: reconnect bugs concentrate in one module
  • interface shrinks per consumer
  • remount hazard visible at the seam
  • leverage: one subscribe, N listeners, tested once

Sequencing: D.14's memoization lands first (planned); this split is the follow-up deepening once Phase D settles — not a blocker.

Problem

AppContent is the state module for the whole Series→Chapter→Page hierarchy; its interface (8 drilled props per route) is as wide as its implementation.

Solution

Move the library slice behind a context seam with a memoized value; routes subscribe to their slice. Depth test applies: if the context interface ends up as wide as the props were, nothing is gained — decide after D.14 lands and measures.

Wins

  • interface per route shrinks to its slice
  • locality: hierarchy bugs in one module
  • leverage: D.6 upload widget reuses the seam
  • risk: wide context = moved shallowness

Problem

The largest module has no internal seams — every concern shares 35 pieces of state, and its coverage exclusion means nothing inside is a test surface.

Solution

During Phase 7, extract only the panels whose extraction ships with tests (LayerPanel, ElementInspector). If tests don't come along, do a pure JSX swap in place — the migration must not become a stealth refactor.

Wins

  • each seam = a new test surface
  • locality: layer bugs leave the core
  • coverage exclusion stops mattering
  • risk: extraction without tests = zero depth

Top recommendation

Start with Candidate 1 — split the notification module, after D.14 lands.

D.14's memoization stops the bleeding; Candidate 1 is the deepening that makes the seam real — transport, list state, and job pub/sub each get an interface worth testing, and the SSE-remount hazard becomes visible at the seam instead of hiding in a 94-line module that every Phase D item depends on.
